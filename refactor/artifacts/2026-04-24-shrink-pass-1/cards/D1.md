# Card D1 - Collapse duplicated format test context helpers

## Equivalence contract

- Inputs covered: test-only callsites in `src/format/markdown.rs` and `src/format/syntax.rs`.
- Ordering preserved: yes; tests would still construct an `OutputContext` immediately before invoking the same render/highlight function.
- Tie-breaking: N/A.
- Error semantics: unchanged; helper construction does not return `Result` or panic.
- Laziness: unchanged; the same `OutputContext` value is materialized per call.
- Short-circuit eval: N/A.
- Floating-point: N/A.
- RNG/hash order: N/A.
- Observable side effects: unchanged; helper construction has no observable side effects beyond producing the context value.
- Public API: unchanged; all code is inside `#[cfg(test)]` modules.

## Proposed edit

Replace, in each target test module:

```rust
fn plain_ctx() -> OutputContext {
    OutputContext::with_mode(OutputMode::Plain)
}

fn json_ctx() -> OutputContext {
    OutputContext::with_mode(OutputMode::Json)
}
```

and the matching `quiet`, `toon`, and `rich` helpers with:

```rust
fn ctx(mode: OutputMode) -> OutputContext {
    OutputContext::with_mode(mode)
}
```

Then update callsites from `&plain_ctx()` to `&ctx(OutputMode::Plain)`, etc.
